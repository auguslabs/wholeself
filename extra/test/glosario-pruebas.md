# Glosario rapido de ambientes y pruebas

- **Ambiente**: conjunto de configuraciones y servicios para ejecutar la app.
- **CI**: integracion continua, pruebas automaticas por cada cambio.
- **CD**: entrega o deploy continuo con controles.
- **Staging**: ambiente que replica produccion.
- **Seed**: datos de prueba generados para la DB.
- **Migracion**: script versionado que cambia el schema.
- **Smoke test**: verificacion minima despues de deploy.
- **E2E**: pruebas que simulan al usuario final.
- **Rollback**: volver a una version anterior estable.
- **Feature flag**: activar o desactivar funcionalidades por configuracion.
